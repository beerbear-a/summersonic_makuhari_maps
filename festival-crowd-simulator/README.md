# Festival Crowd Simulator

SUMMER SONIC 東京会場マップ（ZOZOマリンスタジアム + 幕張メッセ + ビーチ）をモチーフにした、
ドット絵スタイルの2D俯瞰型群衆シミュレーションゲームです。
タイムテーブルは2026年分が未発表のため、**実際に開催された SUMMER SONIC 2025 東京公演
2日目（2025/8/17）のラインナップ**を採用しています（詳細は `src/data/timetable.ts` のコメント参照）。
800人の観客エージェントがタイムテーブルに従って6つのステージ・トイレ・飲食・物販・駅を
フローフィールド経路探索で移動し、混雑がヒートマップで可視化されます。
俯瞰モードではフェス運営側として混雑度・安全度・満足度を観察でき、
歩くモードではプレイヤー自身が観客の一人としてフェスを「体験」できます。

## 起動方法

```bash
npm install
npm run dev
```

ブラウザで表示された URL（通常 http://localhost:5173）を開いてください。

本番ビルドは `npm run build`（型チェック込み）、確認は `npm run preview` です。

## 操作

- **⏸ / ▶ ボタン**: 一時停止・再生
- **1x / 2x / 4x / 8x**: 再生速度の変更（1x で実時間1秒 = シミュレーション2.5分）
- **🗺 俯瞰モード（全体マップ）**: 会場全体を一望。混雑ヒートマップと人流の観察用
- **🎮 歩くモード（詳細マップ）**: 実際に会場を歩いて回る縮尺のフィールドに切替。
  プレイヤーキャラを WASD / 矢印キーで操作し、**Shift でダッシュ**、
  Q / E でズーム調整（1〜4倍）。左上に**ミニマップ**（全体マップ + 現在地）を表示。
  💬が出ている観客や施設の看板に **Space / Z** で話しかけるとセリフが聞けます。
  観客のセリフは状態（鑑賞中・空腹・ストレスなど）で変わり、
  ステージの看板は NOW PLAYING / NEXT を教えてくれます。
  人混みの中では歩く速度が落ちます。もう一度押すと俯瞰モードに戻ります
- **プレイヤー自身もフェスを体験できます**（歩くモード右上のHUD）:
  自分の hunger / toilet / fatigue / HYPE を持ち、FOOD AREA・トイレ・GOODS で
  話しかけると実際にお腹が満たされたりグッズを買ったりできます。
  ステージの客席エリアに立って8秒以上ライブを観ると「参加」として記録され、
  画面がほんのり光でパルスし、人気アクトほど画面が軽くシェイクします。
  サイドパネルの **MY FES** に、食べた回数・トイレ回数・買ったグッズ・
  最前で観たライブの一覧が記録されていきます
- **ゲーム的な要素**: 行動に応じて右上に **SCORE** が伸びていき、
  「初めての現場」「はしごファン」「HYPE MAX」など全8種の**実績**を解除すると
  画面上部にトーストで通知（ファンファーレ付き）。フェスが終わる（全員退場する）と
  会場満足度とプレイヤーの体験を合わせた **S〜Dランクのリザルト画面**が出ます。
  行動の効果音・実績ファンファーレは右上の 🔊 ボタンでオン/オフできます
- フェスは 10:30 開場 → 21:20 閉場。17:00 頃から日が暮れてマップが夜になり、
  ライブ中のステージには照明が灯ります。大トリ ALICIA KEYS は 19:30〜20:50。
  閉場後は全員が海浜幕張駅へ向かい、退場完了でリザルト画面が表示されて終了

## 仕組みの概要

- **座標系**: 論理レイアウト(672x1120)を WORLD_SCALE(=4) 倍した 2688x4480 の
  詳細スケールがワールド座標。俯瞰モードは 0.25 倍で全体を表示する。
  タイルは 32px・キャラクターは 24x40px で「歩いて回れる」縮尺になっている
- **装飾の拡張**: `data/decorations.ts` に種類と座標を足すだけでマップに
  旗・ヤシの木・ベンチ・テント・スピーカー等を配置できる（描画は TileMap 側に集約）。
  会場全体をフェス仕様で装飾済み: エントランス広場と南広場のガーランド（三角旗）、
  駅前大通り・サブエントランス沿いの旗の並木道、各ステージ客席前のクラウドコントロール
  フェンス、ビーチのパラソル、FOOD/GOODSエリアのバルーン、通路のプランター・ゴミ箱など
  （全11種の装飾、`line()`/`barrierRow()`/`barrierCol()` ヘルパーで並べて配置している）
- **時間**: 内部は「分」で管理（12:00 = 720）。1x 再生で実時間1秒 = 2.5分進む
- **経路探索**: 施設・経由地ごとに Dijkstra の距離場（フローフィールド）を起動時に事前計算。
  観客は自分のいるタイルの方向ベクトルに従うだけで建物・水域を迂回して移動する
- **順路追従**: 地形ごとに歩行コストが異なる（順路 < 広場 < 砂浜 << 芝生 < 車道）。
  最短経路が自然と順路・広場・横断歩道沿いになり、芝生や車道のショートカットはほぼしない
- **観客**: hunger / toilet / fatigue / satisfaction / stress を持ち、
  タイムテーブルと興味スコア（人気度×個人補正）で目的地を決定。
  興味が高い人ほどステージ前方に詰め、興味が薄い人は後方で観る。
  見たいアクトがなくなった人は閉場前から少しずつ帰り始める（分散退場）
- **混雑**: 96pxセルで人数をカウント。0-2=空き / 3-6=やや混雑 / 7-12=混雑 / 13+=危険水準。
  混雑セルでは移動速度が最大75%低下し、ストレスが上がる
- **スコア**: 混雑度・安全度は混雑/危険セル数から、満足度は観客平均から算出
- **観客キャラ**: 8x12pxの頭でっかちチビキャラ（丸い目・チーク・前/後/横向き・歩行アニメ・
  髪や肌のバリエーション）。シャツの色が状態を表す:
  白=移動中 / 水色=鑑賞中 / 橙=食事 / 紫=トイレ / 桃=買い物 / 灰=退場
- **タイムテーブル**: SUMMER SONIC 2025 東京2日目（2025/8/17）の実出演アーティスト19組。
  MARINE STAGE の開演時刻・出演順は公開情報のまま、他ステージは実際の出演アーティストを
  ゲームバランス上の近い時間帯に配置（詳細は `data/timetable.ts` 冒頭のコメント）
- **プレイヤーの体験**: 観客と同じ hunger / toilet / fatigue に加え HYPE（満足度相当）を持つ。
  FOOD/トイレ/GOODS の看板に話しかけると実際に数値が変化し、ステージ客席で8秒鑑賞すると
  MY FES に記録される。人気アクトを間近で観るほど画面が揺れて臨場感が増す
- **スコア・実績・リザルト**: SCORE = 食事×15 + グッズ×20 + トイレ×5 + 観たライブ×60 + HYPE。
  `game/Achievements.ts` の条件を満たすと実績トーストが出る（新しい実績は条件を1つ足すだけで追加可）。
  フェス終了時に会場満足度(40%)とプレイヤーのHYPE/体験(60%)を合成してS〜Dランクを算出し、
  リザルト画面に表示する
- **効果音**: `game/Sound.ts` が Web Audio API の発振器でその場合成するため音声ファイル不要。
  会話・行動成功・実績解除・モード切替でそれぞれ違う音を鳴らし、🔊ボタンでミュート可能

## 今後の拡張案

1. **施設配置エディタ**: トイレ・飲食の位置や数をプレイヤーが編集し、1日のスコアで採点するゲームループ化
2. **タイムテーブル編集**: 被り・転換時間を調整して混雑を平準化するパズル要素
3. **群衆の局所回避**: Boids / RVO 的な分離力で押し合い・対向流のすれ違いを表現
4. **入退場の波**: 電車到着に合わせた入場波、規制退場（分散退場）の演出強化
5. **実データ接続**: リポジトリ内の幕張メッセ座標データ（action_map.json）による精密化
6. **イベント演出**: アンコール・ゲリラ豪雨・グッズ列爆発などのランダムイベント
7. **サウンド**: ステージ近くでチップチューンBGMが聞こえるなどの演出
8. **パフォーマンス**: エージェント数を増やす場合は空間ハッシュ＋ParticleContainer 化
