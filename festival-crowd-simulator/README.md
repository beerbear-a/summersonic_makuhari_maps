# Festival Crowd Simulator

SUMMER SONIC 2026 東京会場マップ（ZOZOマリンスタジアム + 幕張メッセ + ビーチ）をモチーフにした、
ドット絵スタイルの2D俯瞰型群衆シミュレーションゲームです。
800人の観客エージェントがタイムテーブルに従って6つのステージ・トイレ・飲食・物販・駅を
フローフィールド経路探索で移動し、混雑がヒートマップで可視化されます。
プレイヤーはフェス運営側として、混雑度・安全度・満足度を観察します。

## 起動方法

```bash
npm install
npm run dev
```

ブラウザで表示された URL（通常 http://localhost:5173）を開いてください。

本番ビルドは `npm run build`（型チェック込み）、確認は `npm run preview` です。

## 操作

- **⏸ / ▶ ボタン**: 一時停止・再生
- **1x / 2x / 4x / 8x**: 再生速度の変更（1x で実時間1秒 = シミュレーション2.5分）
- **🎮 歩くモード**: カメラが6倍ズーム（Q / E で3〜8倍に調整可）になり、
  プレイヤーキャラを WASD / 矢印キーで操作して会場内を自由に歩けます（ポケモン風）。
  💬が出ている観客や施設の看板に **Space / Z** で話しかけるとセリフが聞けます。
  観客のセリフは状態（鑑賞中・空腹・ストレスなど）で変わり、
  ステージの看板は NOW PLAYING / NEXT を教えてくれます。
  人混みの中では歩く速度が落ちます。もう一度押すと俯瞰モードに戻ります
- フェスは 11:00 開場 → 21:30 閉場。17:00 頃から日が暮れてマップが夜になり、
  ライブ中のステージには照明が灯ります。閉場後は全員が海浜幕張駅へ向かい、退場完了で終了

## 仕組みの概要

- **時間**: 内部は「分」で管理（12:00 = 720）。1x 再生で実時間1秒 = 2.5分進む
- **経路探索**: 施設・経由地ごとに Dijkstra の距離場（フローフィールド）を起動時に事前計算。
  観客は自分のいるタイルの方向ベクトルに従うだけで建物・水域を迂回して移動する
- **順路追従**: 地形ごとに歩行コストが異なる（順路 < 広場 < 砂浜 << 芝生 < 車道）。
  最短経路が自然と順路・広場・横断歩道沿いになり、芝生や車道のショートカットはほぼしない
- **観客**: hunger / toilet / fatigue / satisfaction / stress を持ち、
  タイムテーブルと興味スコア（人気度×個人補正）で目的地を決定。
  興味が高い人ほどステージ前方に詰め、興味が薄い人は後方で観る。
  見たいアクトがなくなった人は閉場前から少しずつ帰り始める（分散退場）
- **混雑**: 24pxセルで人数をカウント。0-2=空き / 3-6=やや混雑 / 7-12=混雑 / 13+=危険水準。
  混雑セルでは移動速度が最大75%低下し、ストレスが上がる
- **スコア**: 混雑度・安全度は混雑/危険セル数から、満足度は観客平均から算出
- **観客キャラ**: 6x10pxのドット絵（前/後/横向き・歩行アニメ・髪や肌のバリエーション）。
  シャツの色が状態を表す: 白=移動中 / 水色=鑑賞中 / 橙=食事 / 紫=トイレ / 桃=買い物 / 灰=退場

## 今後の拡張案

1. **施設配置エディタ**: トイレ・飲食の位置や数をプレイヤーが編集し、1日のスコアで採点するゲームループ化
2. **タイムテーブル編集**: 被り・転換時間を調整して混雑を平準化するパズル要素
3. **群衆の局所回避**: Boids / RVO 的な分離力で押し合い・対向流のすれ違いを表現
4. **入退場の波**: 電車到着に合わせた入場波、規制退場（分散退場）の演出強化
5. **実データ接続**: リポジトリ内の幕張メッセ座標データ（action_map.json）による精密化
6. **イベント演出**: アンコール・ゲリラ豪雨・グッズ列爆発などのランダムイベント
7. **サウンド**: ステージ近くでチップチューンBGMが聞こえるなどの演出
8. **パフォーマンス**: エージェント数を増やす場合は空間ハッシュ＋ParticleContainer 化
