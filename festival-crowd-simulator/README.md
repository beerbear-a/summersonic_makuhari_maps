# Festival Crowd Simulator

幕張メッセ風の架空フェス会場を舞台にした、2D俯瞰型の群衆シミュレーションゲーム（MVP）です。
700人の観客エージェントがタイムテーブルに従ってステージ・トイレ・飲食・物販・出口を移動し、
混雑がヒートマップで可視化されます。プレイヤーはフェス運営側として、混雑度・安全度・満足度を観察します。

## 起動方法

```bash
npm install
npm run dev
```

ブラウザで表示された URL（通常 http://localhost:5173）を開いてください。

本番ビルドは `npm run build`（型チェック込み）、確認は `npm run preview` です。

## 操作

- **⏸ / ▶ ボタン**: 一時停止・再生
- **1x / 2x / 4x / 8x**: 再生速度の変更（1x で実時間1秒 = シミュレーション2分）
- フェスは 11:30 開場 → 16:30 閉場。閉場後は全員が EXIT へ向かい、退場完了で終了

## 仕組みの概要

- **時間**: 内部は「分」で管理（12:00 = 720）。1x 再生で実時間1秒 = 2分進む
- **観客**: hunger / toilet / fatigue / satisfaction / stress を持ち、
  タイムテーブルと興味スコア（人気度×個人補正）で目的地を決定
- **混雑**: 1200x800 を 40px グリッドに分割して人数をカウント。
  0-4人=空き / 5-14人=やや混雑 / 15-29人=混雑 / 30人以上=危険水準。
  混雑セルでは移動速度が最大 75% 低下
- **スコア**: 混雑度・安全度は混雑/危険セル数から、満足度は観客平均から算出
- **観客の色**: 白=移動中 / 水色=鑑賞中 / 橙=食事 / 紫=トイレ / 桃=買い物 / 灰=退場

## 今後の拡張案

1. **経路探索**: 現在は直進のみ。壁・ホール区画を作り A* や Flow Field で導線を再現する
2. **施設配置エディタ**: トイレや飲食の位置・数をプレイヤーが編集し、スコアで採点するゲームループ化
3. **タイムテーブル編集**: 被り・転換時間を調整して混雑を平準化するパズル要素
4. **群衆の局所回避**: Boids / RVO 的な分離力で押し合い・対向流のすれ違いを表現
5. **入退場の波**: 電車到着に合わせた入場波、規制退場（分散退場）の再現
6. **実データ接続**: リポジトリ内の幕張メッセ座標データ（action_map.json）を使った実マップ化
7. **パフォーマンス**: エージェント数を増やす場合は空間ハッシュ＋ParticleContainer 化
8. **イベント演出**: アンコール・ゲリラ豪雨・グッズ列爆発などのランダムイベント
