# 03. UI / UX — 画面・操作・ゲームフィール

## 1. 画面フロー

```
タイトル ──→ なまえ入力（初回のみ）──→ プロローグ（自宅）──→ フェス本編
   │                                                        │
   └─ つづきから（ママLINEセーブ）                            ├─ スマホメニュー（ポーズ）
   └─ フリーローム / NG+（解放後）                            ├─ 会話 / カットシーン
                                                             └─ エンディング → リザルト → タイトル
```

## 2. タイトル画面

- ドット絵の夕暮れマクハリ全景（既存マップテクスチャを0.25倍で流用し、観客をゆっくり流す=デモ画面兼用）
- メニュー: はじめから / つづきから / フリーローム / 2日通し券 / せってい
- タイトルロゴはドット組みの日本語ロゴ。BGMはメインテーマのチップチューン

## 3. 本編HUD（歩くモード）

```
┌────────────────────────────────────────────┐
│ ┌ミニマップ┐                    ┌ステータス┐ │
│ │ 全体+現在地│                   │バンドLED×8│ │  ← メモリー獲得状況が常時見える
│ │ 出没NPC点 │                   │4ゲージ    │ │
│ └──────────┘                    │¥残高      │ │
│                                  └──────────┘ │
│                 （フィールド）                  │
│                                                │
│  ┌時計 12:34┐            ┌NOW PLAYING ティッカー┐│
│  └──────────┘            └（次の被り警告もここ）┘│
└────────────────────────────────────────────┘
```

- 現行HUDからの変更点:
  - **SCORE表示を廃止**（内部では計算し、リザルトでのみ見せる。ピラー「数字より思い出」）
  - メモリーバンドのLED×8を常時表示（点灯アニメが最高のご褒美）
  - 画面下に「NOW PLAYING / まもなく開演」ティッカー。**被り**が近いと黄色く点滅
- サイドパネル（現行の右パネル）は**スタッフ端末モード専用**に格下げ。歩行中はゲーム画面に集中させる

## 4. スマホメニュー（ポーズ画面・Tab/Cで開く）

ドット絵のスマホUI。開くと時計停止。タブ構成:

| タブ | 内容 |
| --- | --- |
| マップ | 全体マップ+施設アイコン+自分。クエスト対象の「うわさ」だけ吹き出しで出る（正確なピンは出さない） |
| タイテ | 6ステージ×時間のグリッド。観たものにスタンプ、被りは赤帯。タップで「予定」に星をつけられる（ティッカー連動） |
| バンド | メモリー8つの獲得状況と獲得時写真 |
| もちもの | 8スロット。使う/わたす/せつめい |
| やること | クエストメモ（自動記入・手描き風フォント） |
| アルバム | シャッター先生の写真+実績一覧 |
| LINE | ママとのトーク履歴=セーブ画面。「既読をつける」=セーブ |

## 5. 会話UI（現行を拡張）

- 現行のポケモン風ウィンドウ+タイプライターを維持
- **追加1: 選択肢**（はい/いいえ、最大3択）。選択肢はウィンドウ右下にカーソル付き縦並び
- **追加2: 顔チップ**。名前つきNPCは16×16pxの顔アイコンをウィンドウ左に表示
- **追加3: 感情エフェクト**。頭上に「！」「♪」「💧」の2フレームアニメ（会話開始・驚き・照れ）
- 連打防止・クールダウンは現行仕様を踏襲

## 6. カットシーン演出仕様

- 上下レターボックス（各48px）がスライドイン。HUD非表示・入力はZ送りのみ
- カメラはCutsceneDirectorがworld座標のキーフレームを補間（ズーム含む）
- スキップ: Xボタン長押し1秒（周回者への礼儀）

## 7. フィナーレ演出（最重要シーン・専用仕様）

19:30 ALICIA KEYS開演〜終演〜花火。ここだけはリッチに作る。

1. 開演: 会場全体が一段暗転→ステージだけ点灯→歓声SE→レターボックス30秒
2. 鑑賞中: スタジアムの観客全員が**ペンライト色に発光する2pxドット**を持つ（描画は加算合成の点螢光）。
   曲想（チップチューンのセクション）に合わせて色が波打つ（サイン波で位相をずらすだけで波になる）
3. ウェーブ演出: プレイヤーがZを押すタイミングでジャンプ、周囲のNPCが追従
4. 終演: 3秒の完全な静寂（音を全部止める勇気）→ 花火1発目の「ドン」
5. 花火: パーティクル（打ち上げ→開花→残光）。5発+トゥルー時はミライシーンへ

## 8. リザルト画面（改修）

- 1ページ目: **写真スライドショー**（自動再生・シャッター先生の撮影という体）
- 2ページ目: メモリーバンド 8LEDの点灯結果 → ランク表示（S〜D）
- 3ページ目: 統計（歩いた距離・話した人数・食べたもの・使ったお金）と実績
- 「もう一度プレイ」「タイトルへ」

## 9. 操作系（最終仕様）

| 入力 | 動作 |
| --- | --- |
| WASD/矢印 | 移動 |
| Shift | ダッシュ |
| Z / Space / Enter | 決定・話す・調べる・セリフ送り |
| X / Esc | キャンセル・カットシーンスキップ長押し |
| C / Tab | スマホメニュー |
| Q / E | ズーム（1〜4倍） |
| M | スタッフ端末（俯瞰）切替 |

- ゲームパッド対応は任意（Gamepad API、十字+ABXYマッピング）。設計上はキーボード完結でよい

## 10. ゲームフィール規定

| 事象 | フィードバック |
| --- | --- |
| メモリー獲得 | 時停止0.4秒+白フラッシュ+ジングル+LED点灯アニメ |
| クエスト完了 | トースト（実績と同型・緑） |
| 買い物/食事 | コインSE+お金カウンタがコロコロ回る |
| 混雑密集 | 画面端がわずかに暗くなる+鼓動SE（不快になる一歩手前で止める） |
| 熱中症寸前 | 画面が波打つシェーダー風エフェクト（sin歪み） |
| 豪雨 | 雨粒レイヤー+画面彩度低下、屋根の下では雨音がこもる |
| 夜 | 現行の暗転+照明に加え、施設看板がネオン点灯 |

  画面シェイク強度（切/弱/強）、フルスクリーン
